2. category-selection page

This Category Selection Page has a one time occurance which only appears for the first login. This page is very important for a smooth and hassel free user experience as this gives app an idea of
about the need of the user for which he will use the app. This significantly improves performance as the next page(Home Page)will suggest freelancers based upon this prior selction.
The Category Selection Page contains two distinctive pannels, namely:

1. The Search Panel

The Search Panel allows the user to look for a particular category of he might want freelancers for. This field is linked with the Database and as the user types something, the app automatically suggests some of the categories.
The general working of the sarch bar is shown in the following flowchart:

Once the user finds the desired category , he can select it by simply taping to the category name. This will add the category in the Selected Category Panel

2. The Selected Category panel

This Panel aquires 90% of the screen space. The purpose of this panel is to display all the categories the user have selected. This is done by making a tab for each of the selected category. The category tab also have a dismiss button, which when clicked, that particular category is deleted from the users personal category database.

3. The Done Button

Once the user is satisfied with the categories he will proceed by cliclking the DONE button on the top right corner of the page. This will redirect the user to theHOME Page of the App.

3. HOME page

The home page of the App or the index page of App contains various fields and panels. This page gives user access to all the features in App. These panels and fiel:

1. The Freelancer Profile

This panel shows a the list of freelancers (i.e users in Work mode of th App), through mini profile windows. These profile windows contains:
1. The Name of the Freelancer
2. Ratings of the freelancer
3. The category he works in.
4. A hire button to directly hire the Freelenacer
In this freelancer profile panel the most important section is the HIRE button. This button lets the user to Directly hire that particular freelancer from the project-details page. The user can also view the complete profile of the freelancer by clicking his name.

2. The Post Project Shortcut Button

The key feature of the Freelancer Application is to post project(work) to the various freelancers connected with the app. We have provided a Shortcut button at the bottom right corner of the screen for this task. This post-project button re-directs the user to the project-details page in the app, from where, after entering the project details, user can contact various freelancers.

3. The Menu panel

The Menu panel can be opened by tapping the icon on the top-left corner.

4. Project-Details(1): Hire Directly

In This page the user is asked to fill the project details, or the details of the specific task he needs a freelancer for. The form in this page is constructed to ask only the minimal and yet the most necessary information that a freelancer would need before accepting the job.
These information gathering fields are divided into three sections, namely:
1. The Project Information

This Field deals with the general details of the project, namely:
1. A short Name for the Project.
2. A Brief Description of the project.
3. The category of the project.
4. A hire button to directly hire the Freelenacer

2. Duration of work

This field division consists two input fields: 1.Start 2.End
And a dropdown bar for slection between Hours, Minutes,Days. This form requires the user to give a rough estimation of the time this particular project would consume.

3. Deadline of Work

This panel asks the user to provide a deadline for the project, i.e, a date by which he requires the project to be complete.

4. Post Button

The post button at the top right corner of the screen posts the project to the selected freelancer, in this case, 'Abdullah'.
